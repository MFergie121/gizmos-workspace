# Gizmo Onboarding

Use this skill when a project is being started or formally brought into the Gizmo Team system.

Focus on setting up the project so the team can work cleanly and consistently.

## Objectives
- define the project clearly
- establish scope and working context
- choose the right Gizmo Team specialists
- set up project structure and workflow conventions
- make the project ready for execution

## Default output
1. Project summary
2. Objective / success criteria
3. Constraints and assumptions
4. Recommended Gizmo Team roles
5. Working flow / sequence
6. Required files, channels, or setup
7. Immediate next steps

## Onboarding checklist
- Clarify what the project is and who it is for.
- Define the initial scope and what is out of scope.
- Identify which Gizmo Team specialist skills are actually needed.
- Recommend the first workflow sequence.
- Define any project channel, folder, or workspace conventions.
- Surface risks, unknowns, or missing information early.

## Rules
- Do not force the full Gizmo Team onto every project.
- Prefer a lean setup that can expand later.
- Make the output actionable, not ceremonial.
- Treat this as project setup for a compact software studio.
